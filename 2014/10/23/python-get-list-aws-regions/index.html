Listing AWS regions with Python

Date Thu 23 October 2014 Tags ansible / aws

With Amazon's announcement of the new Frankfurt AWS region (eu-central-1), I've gone back and reworked an couple of old Python scripts that had region lists hardcoded in them.

I stopped hardcoding this a few years ago, and replaced it with this snippet of code. You can just run it as a Python file, or from the command line. It does require the Python package boto version 2.34.0 or higher to be installed.

I also strip out the China region and the US Govcloud region, since I don't have services in either location, and I sort the list as well.

print_aws_regions.py

from boto import ec2

REGIONS = sorted([r.name for r in ec2.regions() if r.name not in ["cn-north-1", "us-gov-west-1"]])

for r in REGIONS:
    print r

Output:

ap-northeast-1
ap-southeast-1
ap-southeast-2
eu-central-1
eu-west-1
sa-east-1
us-east-1
us-west-1
us-west-2

Command line version

The variables for Ansible are defined in group_vars/vagrant, which pulls them in from the two environment variables set on my Mac.

python -c 'from boto import ec2;REGIONS = sorted([r.name for r in ec2.regions() if r.name not in ["cn-north-1", "us-gov-west-1"]]); print REGIONS'

Output:

[u'ap-northeast-1', u'ap-southeast-1', u'ap-southeast-2', u'eu-central-1', u'eu-west-1', u'sa-east-1', u'us-east-1', u'us-west-1', u'us-west-2']
